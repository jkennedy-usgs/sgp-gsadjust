USGS Southwest Gravity Program

NWIS example

 
This example demonstrates the comparison of gravity-change data with USGS National Water Information System (NWIS) data. After processing (network adjustment) gravity data, results can be compared with groundwater-level change in map view or in a time-series and specific yield view.
 
1. Open workspace
 
First, load a workspace with multiple surveys in a study area with groundwater-level data in NWIS. An example workspace, test_data/NWIS_example is included, from Kennedy and others (2022).
 
Once open, perform the network adjustment for all surveys (Ctrl+1 or )
 
 
2. Show NWIS data in map view
 
Open a map-view window with Tools > Compute gravity change > Map button.
 
For any particular time period (selected with the "Incremental" and "Change relative to..." buttons and the slider below the map) checking the "Show NWIS water levels" box will:
-Search NWIS for groundwater sites within the map view and the date range
-Plot results on the map using a brown to green color scale and square markers
-Show the NWIS sites in a table below the map.
 
 
Like with gravity data, the station name is shown on the map when a groundwater-level station is clicked. Groundwater-level data are shown in the table at the bottom of the window. Groundwater levels can be excluded from the map by unchecking them in the table. Because groundwater levels are retrieved from NWIS each time the map is updated, it may be useful to fine-tune the other elements of the map before checking the "Show NWIS water levels" box (a similar consideration applies to the "Show basemap" option).
 
By default, only groundwater levels from sites with a daily value on the beginning and ending dates of the gravity interval are shown (typically, continuous sites with transducers). Groundwater levels can be interpolated by specifying a maximum interpolation window in the "Date window (days)" box. If the interval between groundwater levels that bracket the start or end dates is less than this window, a value is interpolated on the date of the gravity measurement. For example, in the graphic above, the start date of the gravity-change interval is 2018-12-10. If the Date window was set to 30 days, a groundwater-level measurement between 2018-11-10 and 2018-12-10 is required, and another between 2018-12-10 and 2019-01-09, in order to interpolate a groundwater level on 2018-12-10. A similar 30-day window is required around the end date of the gravity-change interval (2019-04-22). Although increasing the date window will increase the amount of data shown on the map, there is added risk with longer windows that the groundwater-level change is not linear during that period.
 
The hydrograph for any station can be displayed by clicking the Hydrograph button in the table of groundwater levels at the bottom of the window. This displays the period of record for the station, with daily values plotted as a blue line and discrete groundwater-level measurements (tapedowns) plotted as black dots. The timing of the gravity surveys in the map window is shown by vertical black lines.
 
 
3. Plot time series and specific yield
 
For a station-specific analysis, select "Plot dg vs. NWIS gwl" on the Tools menu.
 
 
Start by selecting a gravity station from the dropdown box. For this example, station JCLI works well. A table shows the observed gravity values at that station. To find a corresponding groundwater level site, click the magnifying box at the right of the "NWIS Station ID" text box. This returns a list of nearby stations sorted by distance.
 
For this example, station 350545106335903, which has the upper-most screened interval at this site, has the best correlation between gravity and groundwater levels. Only one groundwater-level station can be plotted at a time, but multiple plot windows can be open at once. Choose the ...5903 station by checking the checkbox, then click Ok to show the time-series and specific yield plots:
 
 
Various options are available for fine-tuning these plots:
 
  • Interpolate gaps less than (days): This has the same functionality as the map viewer: if there is a gap between measurements less than this value (like with discrete tapedown data), a value at the time of the gravity measurement will be linearly interpolated.
     
  • Time lag for groundwater levels (days): For the purpose of calculating specific yield, this compares the gravity data with the groundwater-level measurement from n days earlier. This accounts for the effect where head changes propagate quicker than storage changes. Mainly useful when there is continuous (i.e., daily) groundwater-level data.
     
  • Find optimal time lag (maximize r2): This varies the above parameter (time lag) between +/- 90 days to find the value that maximizes r2.
     
  • Filter groundwater-level data: This applies a Hanning low-pass filter to the groundwater-level data. It is potentially useful when there is high-frequency pumping noise in the groundwater-level data.
     
  • Window (days): Filter length when the above option is checked.
 
  • Rising/falling groundwater levels only: For the specific yield analysis, only groundwater-levels that are higher/lower than the previous value are included.
